# Спецификация решения

## Назначение

Изделие предназначено для измерения уровня топлива в баке транспортного средства и передачи данных в бортовую телематику или мобильное приложение. Датчик должен работать в условиях вибрации, перепадов температуры, электромагнитных помех и всплесков уровня топлива при движении.

## Функциональные требования

- измерение уровня топлива по емкостному принципу;
- хранение тарировочной таблицы для конкретного бака;
- поддержка расширенной тарировочной таблицы с параметрами уровня, температуры, долива и слива;
- выбор типа топлива через сервисный интерфейс;
- измерение температуры топлива;
- измерение температуры воздуха в головной части датчика;
- фильтрация шумов, плеска и паразитных выбросов;
- передача данных по `BLE`;
- сервисная настройка, диагностика и обновление параметров через `USB`;
- совместимость со встроенным телематическим блоком и Android-приложением;
- возможность дальнейшей отправки данных на сервер `РНИС`.

## Основные выходные данные

- относительный уровень топлива, `%`;
- расчетный объем топлива, `л`;
- температура топлива, `°C`;
- температура воздуха, `°C`;
- статус достоверности измерения;
- режим изменения уровня: стабильно, долив, слив;
- номер профиля топлива;
- диагностические признаки:
  - обрыв или короткое замыкание измерительного канала;
  - выход температуры за допустимые пределы;
  - необходимость повторной тарировки;
  - сервисный режим.

## Сценарии применения

1. Телематический блок периодически считывает данные по `BLE` и пересылает их в транспортный сервер.
2. Android-приложение подключается к датчику рядом с машиной для настройки, проверки и журналирования.
3. Сервисный инженер подключает `USB`, запускает тарировку и выбирает тип топлива.

## Ограничения проекта

- конструктив датчика должен соответствовать классической схеме емкостного топливного зонда;
- головная часть должна быть защищена от влаги, паров топлива и механических воздействий;
- пользовательские операции тарировки должны выполняться без вскрытия корпуса электроники;
- алгоритм должен допускать отдельные настройки реакции на доливы и сливы;
- данные по `BLE` должны быть пригодны как для штатного телематического блока, так и для мобильного приложения.
