# Профиль топливной системы машины

## Назначение

Для многобаковой машины нужен не только набор ДУТ, но и настраиваемый профиль топливной системы. Именно он описывает:

- какие баки физически установлены;
- какие магистрали их соединяют;
- откуда берет топливо каждый агрегат;
- куда приходит обратка;
- есть ли перекачка между баками;
- какие клапаны и насосы меняют маршрут топлива.

Без этого профиля даже хорошие данные `ДУТ` остаются неоднозначными.

## Где хранится профиль

Профиль должен храниться как отдельная сущность конфигурации:

- в `БНСО`;
- на сервере;
- частично в памяти `ДУТ`, если это нужно для локальной логики.

Лучше считать `БНСО` главным владельцем профиля машины, потому что именно он видит все баки и все агрегаты одновременно.

## Что входит в профиль

### 1. Общие параметры машины

- `vehicle_net_id`
- `system_profile_id`
- `system_model_revision`
- тип машины
- тип эксплуатации
- число баков
- число потребителей
- число управляемых маршрутов

### 2. Описание баков

Для каждого бака:

- `tank_id`
- имя бака
- тип: основной / вспомогательный / расходный / дневной
- установленный `sensor_id`
- геометрический профиль
- полезный объем
- признак ведущего бака
- признак участия в общей сумме топлива

### 3. Описание магистралей

Для каждой магистрали:

- `line_id`
- тип: подача / обратка / балансировка / перекачка
- `source_tank_id`
- `target_tank_id`
- номинальное направление потока
- признак постоянно открытой линии
- признак управляемой линии

### 4. Источники питания агрегатов

Для каждого агрегата:

- `consumer_id`
- имя агрегата
- тип потребителя
- основной источник топлива
- допустимый набор баков
- приоритет баков
- признак использования обратки

### 5. Обратка

Для каждого потребителя, у которого есть возврат топлива, нужно настроить:

- есть ли обратка вообще;
- в какой бак возвращается топливо по умолчанию;
- возможна ли обратка в другой бак;
- зависит ли маршрут обратки от клапанов или режима работы.

### 6. Перекачка между баками

Для каждого сценария перекачки:

- `transfer_id`
- `source_tank_id`
- `target_tank_id`
- какой насос отвечает за перекачку;
- какие клапаны должны быть открыты;
- признак автоматической или ручной перекачки;
- событие или сигнал активации.

### 7. Клапаны и насосы

Для каждого управляемого элемента:

- `actuator_id`
- тип: клапан / насос / переключатель
- логическое имя
- связанные баки и линии
- логика управления
- источник телеметрического сигнала

## Минимальные настройки профиля

Если начинать с упрощенного профиля, нужно хотя бы:

- список баков;
- привязку `sensor_id -> tank_id`;
- привязку `consumer_id -> source_tank_id`;
- наличие или отсутствие обратки;
- наличие или отсутствие перекачки;
- участие бака в общей сумме топлива.

## Расширенные настройки профиля

Для более сложной техники профиль должен поддерживать:

- несколько допустимых баков для одного агрегата;
- переключаемый маршрут подачи;
- переключаемый маршрут обратки;
- зависимость маршрута от состояния клапанов;
- режимы, в которых источник топлива меняется автоматически;
- исключение отдельных баков из управленческой аналитики.

## Что должен уметь сервис настройки

Через приложение или сервисный интерфейс пользователь должен иметь возможность:

- создать профиль машины;
- добавить, удалить и редактировать баки;
- добавить и редактировать потребителей;
- описать магистрали и обратку;
- описать логику перекачки;
- описать клапаны и насосы;
- проверить профиль на логические ошибки;
- сохранить профиль и повысить `system_model_revision`.

## Проверки согласованности профиля

Перед сохранением система должна проверять:

- все ли баки имеют уникальный `tank_id`;
- нет ли двух датчиков на один и тот же бак без явного разрешения;
- все ли потребители имеют хотя бы один источник топлива;
- не ведет ли обратка в несуществующий бак;
- не образует ли перекачка недопустимую петлю;
- участвуют ли только валидные баки в общей сумме топлива.

## Практический результат

Правильно настроенный профиль позволяет:

- суммировать топливо по всем бакам машины;
- корректно относить расход к агрегатам;
- отличать штатный расход от перекачки;
- отличать переток от слива;
- строить отчет по работе машины, а не просто график уровня.
