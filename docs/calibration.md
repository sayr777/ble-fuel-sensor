# Тарировка и сервис

## Что настраивается через USB

- `installation_id` или `vehicle_net_id` машины;
- серийный номер датчика;
- `sensor_id`;
- `tank_id`;
- профиль топливной системы машины;
- профиль топлива;
- высота активной части зонда;
- таблица точек тарировки бака;
- коэффициенты температурной компенсации;
- пороги распознавания долива;
- пороги распознавания слива;
- скорость подтверждения долива;
- скорость подтверждения слива;
- параметры сглаживания стабильного режима;
- период измерения и публикации;
- параметры BLE-идентификации.

Поля `installation_id / vehicle_net_id`, `sensor_id` и `tank_id` должны считаться обязательными параметрами первичной привязки. Именно через `USB` сервис должен выполнять:

- назначение датчика конкретной машине;
- привязку датчика к конкретному баку;
- проверку отсутствия конфликта идентификаторов внутри парка;
- фиксацию идентификаторов в энергонезависимой памяти.

Если машина многобаковая, отдельно должны настраиваться:

- состав баков;
- привязка датчика к баку;
- привязка потребителей к бакам;
- обратка;
- перекачка;
- управляемые клапаны и насосы.

## Рекомендуемый процесс тарировки

1. Подключить датчик к сервисному приложению по `USB`.
2. Выбрать тип топлива из справочника.
3. Выполнить базовую проверку пустого бака.
4. Заполнить бак по контрольным точкам и зафиксировать измеренные значения.
5. Сохранить таблицу пересчета "емкость -> литры".
6. Настроить температурные коэффициенты и профиль топлива.
7. Настроить реакцию алгоритма на доливы и сливы.
8. Проверить итоговую кривую и допуски.
9. Зафиксировать тарировку в энергонезависимой памяти.

## Что должно видеть сервисное приложение

- сырое значение измерительного канала;
- отфильтрованное значение;
- температура топлива;
- температура воздуха;
- итоговый уровень, `%`;
- объем, `л`;
- текущий режим: стабильно, долив, слив;
- статус достоверности;
- предупреждение о шумном сигнале;
- предупреждение о необходимости повторной тарировки.

## Замечания по топливам

Профиль топлива нужен для выбора поправок и допустимых рабочих диапазонов. На старте проекта достаточно заложить поддержку:

- бензин;
- дизель;
- керосин;
- пользовательский профиль.

## Смысл расширенной тарировочной таблицы

В этом проекте тарировка не должна ограничиваться одной линейной зависимостью. Полноценная таблица должна хранить:

- геометрию бака и пересчет в литры;
- температурную зависимость;
- параметры устойчивого состояния;
- отдельные правила для долива;
- отдельные правила для слива.

Именно эта часть делает датчик полезным для реальной эксплуатации и телематической аналитики.
