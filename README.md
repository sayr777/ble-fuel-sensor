# BLE датчик уровня топлива на nRF52840

Проект BLE-датчика уровня топлива для установки в топливный бак по классической емкостной схеме. Изделие строится на `nRF52840`, измеряет уровень топлива, температуру топлива и температуру воздуха, выполняет цифровую осечку шумов и передает данные по `BLE` во встроенный телематический блок автомобиля или в Android-приложение с последующей отправкой на сервер `РНИС`.

Главная особенность проекта - расширенная тарировочная таблица, которая описывает не только базовый пересчет измерения в уровень и литры, но и позволяет настраивать температурную зависимость, поведение уровня при доливах и сливах, а также логику сглаживания переходных процессов.

## Базовая идея изделия

- измерительный элемент представляет собой погружной емкостной датчик;
- тарировка и сервис выполняются через `USB`;
- оператор может выбрать тип топлива и профиль пересчета;
- основой вычислений служит тарировочная таблица с дополнительными параметрами коррекции;
- прошивка учитывает температуру топлива и температуру воздуха;
- измерение проходит через фильтрацию помех, всплесков и дрожания уровня;
- алгоритм различает стабильное состояние, долив и слив;
- данные передаются по `BLE` в двух сценариях:
  - во встроенный телематический блок автомобиля;
  - в Android-приложение для настройки, диагностики и публикации данных на `РНИС`.

## Предлагаемая аппаратная база

- SoC: `nRF52840`
- BLE: встроенный радиоблок `2.4 GHz`
- сервисный интерфейс: `USB`
- измерение уровня: емкостной зонд + аналоговый фронтенд
- датчики температуры: канал топлива + канал воздуха
- питание: бортовая сеть через локальный источник питания и защиту
- монтаж: герметичный корпус головной части, погружная измерительная трубка

## Платформы для прототипа

Для быстрого старта можно использовать готовые отладочные платы на `nRF52840`:

- `Promicro NRF52840 Development Board Compatible with nice!nano V2.0` - удобный вариант для сервисной прошивки, USB-отладки и раннего макета;
- `Super 52840 NRF52840 Bluetooth 5.0 Module Development Board Compatible with Seeed XIAO BLE` - компактный вариант для лабораторного прототипа и проверки компоновки.

Для MVP это позволяет вынести разработку прошивки и BLE-обмена вперед, а измерительный емкостной тракт, питание и температурные датчики оформить отдельной платой или макетом. Для серийного изделия позже логично перейти на собственную плату в головной части датчика.

## Ключевые особенности решения

- работа с несколькими видами топлива;
- многопараметрическая тарировочная таблица;
- температурная компенсация измерительного канала;
- отдельное измерение температуры топлива и температуры воздуха;
- раздельные настройки поведения при доливах и сливах;
- алгоритм подавления шумов и резких выбросов;
- сервисная настройка без разборки устройства через `USB`;
- диагностический доступ с Android;
- управленческая аналитика расхода по движению, технологическому циклу и холостому ходу;
- интеграция с телематикой и `РНИС`.

## Следующий этап

Текущий репозиторий - это рабочий каркас проекта на основе структуры `ble-tag-e73`, но уже под датчик уровня топлива. Следующий практический шаг - зафиксировать электрическую схему измерительного фронтенда и формат производственного протокола обмена с телематическим блоком.
