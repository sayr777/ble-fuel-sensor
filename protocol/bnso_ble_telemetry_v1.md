# BNSO BLE Telemetry Profile v1

## Назначение

Этот профиль используется для обмена между `ДУТ` и `БНСО` без установления BLE-соединения. Датчик периодически публикует короткий телематический кадр, а `БНСО` принимает его в фоне.

## Транспорт

- режим: `BLE advertising`;
- тип полезной нагрузки: `Manufacturer Specific Data`, `AD Type = 0xFF`;
- в прототипе допускается использовать тестовый `Company ID = 0xFFFF`;
- в серийном варианте должен использоваться реальный зарегистрированный `Company ID` производителя или интегратора.

## Рекомендуемое поведение эфира

- период штатной публикации: `1000 мс`;
- при изменении состояния `level_state` рекомендуется передать burst из `3` кадров с интервалом `100...200 мс`;
- если измерение недостоверно, кадр все равно передается, но с соответствующими `quality_flags`.

## Формат manufacturer data

| Смещение | Размер, байт | Поле |
| --- | ---: | --- |
| 0 | 1 | `protocol_version` |
| 1 | 1 | `message_type = 0x01` |
| 2 | 2 | `sensor_id` |
| 4 | 1 | `tank_id` |
| 5 | 1 | `system_model_revision` |
| 6 | 1 | `sequence` |
| 7 | 2 | `fuel_level_percent_x10` |
| 9 | 2 | `fuel_volume_l_x10` |
| 11 | 2 | `fuel_temp_c_x10` |
| 13 | 2 | `air_temp_c_x10` |
| 15 | 2 | `tilt_pitch_deg_x10` |
| 17 | 2 | `tilt_roll_deg_x10` |
| 19 | 1 | `level_state` |
| 20 | 1 | `validity_status` |
| 21 | 1 | `confidence_percent` |
| 22 | 2 | `consumer_mask` |
| 24 | 2 | `quality_flags` |
| 26 | 1 | `fuel_profile` |
| 27 | 1 | `calibration_revision` |
| 28 | 2 | `crc16` |

Итоговая длина прикладной части кадра: `30 байт`.

## Алгоритм приема на стороне БНСО

1. Отфильтровать рекламные пакеты по `Company ID`.
2. Проверить `protocol_version`.
3. Проверить `CRC16`.
4. Найти датчик по `sensor_id`.
5. Сопоставить пакет с конкретным `tank_id`.
6. Отбросить пакет, если `sequence` совпадает с уже обработанным.
7. Сохранить телеметрию и время приема.
8. Передать данные в основной канал телематики.

## Таймауты и достоверность

- если новых кадров нет более `5 с`, `БНСО` должен пометить канал как "нет свежих данных";
- если новых кадров нет более `30 с`, `БНСО` должен пометить датчик как "не в сети";
- если `quality_flags` не содержит бит валидности, `БНСО` должен сохранить пакет как диагностический, но не использовать его как достоверный уровень топлива.

## Минимальный набор данных, который БНСО должен передавать выше

- время приема пакета;
- `sensor_id`;
- `tank_id`;
- `system_model_revision`;
- уровень топлива;
- объем топлива;
- температура топлива;
- температура воздуха;
- продольный уклон;
- поперечный уклон;
- `level_state`;
- `validity_status`;
- `confidence_percent`;
- `consumer_mask`;
- `quality_flags`;
- `fuel_profile`;
- `calibration_revision`.
