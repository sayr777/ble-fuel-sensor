# Protocol v1

## Назначение

Этот документ фиксирует общие правила `v1` для всех профилей взаимодействия датчика:

- кодирование чисел;
- общие перечисления;
- флаги качества;
- типы сообщений и команд;
- стандартные статусы ответов.

## Общие правила

- `protocol_version = 0x01`;
- все многобайтные поля кодируются в `Little Endian`;
- отрицательные температуры кодируются в `int16`;
- если значение недоступно, для знаковых полей используется `0x8000`, для беззнаковых полей `0xFFFF`;
- контроль целостности прикладного пакета выполняется с помощью `CRC16-CCITT`, начальное значение `0xFFFF`, полином `0x1021`.

## Базовый набор телеметрических полей

| Поле | Размер, байт | Формат | Комментарий |
| --- | ---: | --- | --- |
| `protocol_version` | 1 | `u8` | Версия протокола |
| `message_type` | 1 | `u8` | Тип сообщения |
| `sensor_id` | 2 | `u16` | Идентификатор ДУТ |
| `sequence` | 1 | `u8` | Счетчик кадров |
| `fuel_level_percent_x10` | 2 | `u16` | Уровень топлива в `0.1 %` |
| `fuel_volume_l_x10` | 2 | `u16` | Объем топлива в `0.1 л` |
| `fuel_temp_c_x10` | 2 | `i16` | Температура топлива в `0.1 °C` |
| `air_temp_c_x10` | 2 | `i16` | Температура воздуха в `0.1 °C` |
| `level_state` | 1 | `u8` | Стабильно, долив или слив |
| `quality_flags` | 2 | `u16` | Биты статуса |
| `fuel_profile` | 1 | `u8` | Профиль топлива |
| `calibration_revision` | 1 | `u8` | Ревизия тарировочной таблицы |
| `crc16` | 2 | `u16` | Контроль целостности |

## Типы сообщений

| Код | Имя | Назначение |
| --- | --- | --- |
| `0x01` | `TELEMETRY` | Короткая телеметрия для БНСО |
| `0x02` | `DIAGNOSTIC` | Расширенная диагностика |
| `0x10` | `APP_REQUEST` | Команда от мобильного приложения |
| `0x11` | `APP_RESPONSE` | Ответ датчика мобильному приложению |
| `0x12` | `APP_EVENT` | Асинхронное событие для приложения |

## Состояние уровня

| Код | Имя | Смысл |
| --- | --- | --- |
| `0x00` | `STABLE` | Уровень стабилен |
| `0x01` | `REFILL` | Обнаружен долив |
| `0x02` | `DRAIN` | Обнаружен слив |

## Флаги качества

| Бит | Маска | Значение |
| --- | --- | --- |
| 0 | `0x0001` | Измерение валидно |
| 1 | `0x0002` | Активна фильтрация всплеска |
| 2 | `0x0004` | Значение удержано антидребезгом |
| 3 | `0x0008` | Ошибка датчика температуры топлива |
| 4 | `0x0010` | Ошибка датчика температуры воздуха |
| 5 | `0x0020` | Ошибка емкостного канала |
| 6 | `0x0040` | Сервисный режим |
| 7 | `0x0080` | Применена температурная коррекция |
| 8 | `0x0100` | Применен профиль долива или слива |
| 9 | `0x0200` | Требуется повторная тарировка |

## Профили топлива

| Код | Значение |
| --- | --- |
| `0x00` | Не задан |
| `0x01` | Бензин |
| `0x02` | Дизель |
| `0x03` | Керосин |
| `0x7F` | Пользовательский профиль |

## Коды статуса для ответов приложения

| Код | Имя | Смысл |
| --- | --- | --- |
| `0x00` | `OK` | Команда выполнена |
| `0x01` | `ERR_UNKNOWN_OPCODE` | Неизвестная команда |
| `0x02` | `ERR_BAD_LENGTH` | Некорректная длина полезной нагрузки |
| `0x03` | `ERR_BAD_STATE` | Команда недопустима в текущем состоянии |
| `0x04` | `ERR_BAD_VALUE` | Поле содержит недопустимое значение |
| `0x05` | `ERR_AUTH_REQUIRED` | Нужна авторизация |
| `0x06` | `ERR_CRC` | Ошибка контрольной суммы |
| `0x07` | `ERR_STORAGE` | Ошибка памяти |
| `0x08` | `ERR_BUSY` | Устройство занято |

## Команды мобильного приложения

| Код | Имя | Назначение |
| --- | --- | --- |
| `0x01` | `HELLO` | Чтение версии и возможностей |
| `0x02` | `AUTH` | Авторизация для записи |
| `0x10` | `READ_TELEMETRY` | Получить снимок телеметрии |
| `0x11` | `READ_DIAGNOSTICS` | Получить расширенную диагностику |
| `0x20` | `READ_CONFIG` | Прочитать конфигурацию |
| `0x21` | `WRITE_CONFIG` | Изменить конфигурацию |
| `0x30` | `READ_CAL_INFO` | Прочитать метаданные тарировки |
| `0x31` | `READ_CAL_POINT` | Прочитать точку тарировочной таблицы |
| `0x32` | `WRITE_CAL_POINT` | Записать точку тарировочной таблицы |
| `0x33` | `DELETE_CAL_POINT` | Удалить точку таблицы |
| `0x34` | `READ_TEMP_COMP` | Прочитать температурные коэффициенты |
| `0x35` | `WRITE_TEMP_COMP` | Записать температурные коэффициенты |
| `0x36` | `READ_EVENT_FILTERS` | Прочитать параметры долива и слива |
| `0x37` | `WRITE_EVENT_FILTERS` | Записать параметры долива и слива |
| `0x38` | `COMMIT_CALIBRATION` | Применить и проверить тарировку |
| `0x39` | `SAVE_TO_NVM` | Сохранить все изменения |
| `0x3A` | `REBOOT` | Перезапуск датчика |
