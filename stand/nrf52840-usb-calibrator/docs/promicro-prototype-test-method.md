# Методика тестирования и проверки двухплатного прототипа ProMicro

## Назначение

Методика описывает, как проверять ранний прототип на двух платах `ProMicro NRF52840`:

- плата `ДУТ`;
- плата `БНСО`.

Цель проверки - убедиться, что базовая телеметрия, сценарии событий и стендовая логика работают до появления собственной цифровой платы.

## Состав проверяемого комплекта

- `ProMicro NRF52840 #1` с прошивкой `ДУТ`;
- `ProMicro NRF52840 #2` с прошивкой `БНСО`;
- ПК с двумя `USB` портами;
- `Arduino IDE` или другой инструмент прошивки;
- два `Serial Monitor` окна;
- по возможности Android-смартфон для отдельной BLE-проверки.

## Предварительные условия

Перед стартом испытаний должно быть подтверждено:

- обе платы определяются в системе по `USB`;
- на плату `ДУТ` загружен скетч `promicro_dut_proto`;
- на плату `БНСО` загружен скетч `promicro_bnso_emulator`;
- на стороне `ДУТ` доступен ввод команд через `Serial`;
- на стороне `БНСО` включен вывод журнала приема пакетов.

## Проверка 1. Базовая загрузка

Шаги:

1. Подключить обе платы к ПК.
2. Открыть `Serial Monitor` для `ДУТ`.
3. Открыть `Serial Monitor` для `БНСО`.

Критерии приемки:

- `ДУТ` выводит стартовое сообщение;
- `БНСО` выводит сообщение о запуске сканирования;
- платы не перезагружаются циклически;
- USB-соединение стабильно не менее `3` минут.

## Проверка 2. Телеметрия `ДУТ -> БНСО`

Шаги:

1. На стороне `ДУТ` выполнить `status`.
2. Дождаться нескольких рекламных пакетов на стороне `БНСО`.

Критерии приемки:

- `БНСО` принимает пакеты без постоянных ошибок;
- значения `sensor_id`, `level`, `volume`, `fuel_temp`, `air_temp` декодируются корректно;
- `sequence` растет монотонно;
- при повторном приеме одного и того же пакета возможна дедупликация.

## Проверка 3. Смена сценариев уровня

Шаги:

1. Выполнить на `ДУТ` команду `set state stable`.
2. Выполнить `set state refill`.
3. Выполнить `set state drain`.

Критерии приемки:

- `БНСО` видит корректную смену `level_state`;
- при `burst` или быстрой смене режима `БНСО` не зависает;
- лог содержит понятную привязку режима к кадру.

## Проверка 4. Изменение уровня и объема

Шаги:

1. Выполнить `set level 574`.
2. Выполнить `set volume 3210`.
3. Выполнить `set level 615`.
4. Выполнить `set volume 3440`.

Критерии приемки:

- новые значения появляются в логах `БНСО`;
- значения передаются без перепутывания полей;
- не наблюдается обнуления других параметров.

## Проверка 5. Температуры

Шаги:

1. Выполнить `set fueltemp 185`.
2. Выполнить `set airtemp 223`.
3. Изменить температуры на отрицательные или граничные значения.

Критерии приемки:

- знаковые температуры декодируются правильно;
- значения приходят именно как `0.1 °C`;
- при изменении температуры остальные поля остаются валидными.

## Проверка 6. Флаги качества

Шаги:

1. Выполнить `set flags 0x0001`.
2. Выполнить `set flags 0x0181`.
3. Выполнить `set flags 0x0201`.

Критерии приемки:

- `БНСО` показывает измененные `quality_flags`;
- битовая маска не искажается;
- критические флаги можно визуально распознать в журнале.

## Проверка 7. Burst и устойчивость приема

Шаги:

1. На `ДУТ` выполнить `burst`.
2. Повторить burst несколько раз подряд.

Критерии приемки:

- `БНСО` принимает серию пакетов;
- не возникает зависаний и постоянных `crc_error`;
- после burst система возвращается в штатный режим.

## Проверка 8. Длительный прогон

Шаги:

1. Оставить стенд работать не менее `30` минут.
2. Один раз в `5` минут менять одно из значений на `ДУТ`.

Критерии приемки:

- не наблюдается утечки связи;
- USB-порты не переподключаются самопроизвольно;
- BLE-прием на стороне `БНСО` не деградирует;
- команда `status` на `ДУТ` остается доступной.

## Журнал испытаний

Для каждого прогона нужно фиксировать:

- дату и время;
- версии скетчей `ДУТ` и `БНСО`;
- `sensor_id`;
- номер сценария;
- ожидаемый результат;
- фактический результат;
- замечания и дефекты.

## Минимальный чек-лист приемки

- стенд на двух платах запускается стабильно;
- телеметрия принимается на `БНСО`;
- поля протокола декодируются корректно;
- `level_state` переключается правильно;
- температуры и флаги передаются без ошибок;
- серийная работа не менее `30` минут проходит без отказа.
