# Настройка стенда на базе прототипов ProMicro NRF52840

## Назначение

Этот стенд нужен для ранней отладки проекта без собственной цифровой платы. Он позволяет параллельно разрабатывать:

- прошивку `ДУТ`;
- BLE-протокол для `БНСО`;
- сервисный интерфейс по `USB`;
- мобильный сценарий диагностики;
- логику тарировки и фильтрации.

## Рекомендуемая конфигурация стенда

### Минимальный вариант

- `ProMicro NRF52840 #1` - плата `ДУТ`;
- `ProMicro NRF52840 #2` - плата `БНСО`;
- ПК с `USB` - сервисный хост;
- внешний макет измерительного фронтенда;
- имитатор емкостного канала или эталонный датчик;
- два датчика температуры или их эмуляторы.

### Рабочий вариант для протокола

- `ProMicro NRF52840 #1` - `ДУТ`;
- `ProMicro NRF52840 #2` - эмулятор `БНСО`;
- Android смартфон - проверка `GATT` профиля;
- ПК - логирование, тарировка и прошивка.

### Расширенный вариант

- `ProMicro NRF52840 #1` - `ДУТ`;
- `ProMicro NRF52840 #2` - `БНСО`;
- `ProMicro NRF52840 #3` - генератор тестовых сценариев или BLE-снифферный помощник;
- лабораторный источник питания;
- макетная плата с коммутируемыми имитаторами уровня и температуры.

## Распределение ролей между платами

### Плата `ДУТ`

Должна реализовывать:

- сбор измерений;
- применение тарировочной таблицы;
- определение режима `stable/refill/drain`;
- BLE advertising для `БНСО`;
- `GATT` сервис для Android;
- `USB` сервис для тарировки.

### Плата `БНСО`

Должна реализовывать:

- фоновое сканирование BLE;
- фильтрацию по `manufacturer data`;
- разбор телематического кадра;
- проверку `CRC` и `sequence`;
- вывод телеметрии в `USB CDC` лог или в тестовый протокол на ПК.

## Arduino-скетчи для стенда

В проекте предусмотрен минимальный набор скетчей для `Arduino IDE`:

- [../firmware/arduino/promicro_dut_proto/promicro_dut_proto.ino](../firmware/arduino/promicro_dut_proto/promicro_dut_proto.ino) - прототип `ДУТ`;
- [../firmware/arduino/promicro_bnso_emulator/promicro_bnso_emulator.ino](../firmware/arduino/promicro_bnso_emulator/promicro_bnso_emulator.ino) - эмулятор `БНСО`.

Оба скетча рассчитаны на две одинаковые платы `ProMicro NRF52840` и позволяют без собственной цифровой платы проверить:

- BLE advertising профиль для `БНСО`;
- базовую телеметрию по уровню и температурам;
- смену режима `stable/refill/drain`;
- обмен по `USB Serial`.

## Настройка Arduino IDE

Рекомендуемая среда:

- `Arduino IDE 2.x`;
- core `Adafruit nRF52 Boards`;
- библиотека `Bluefruit52Lib`.

### Базовые шаги

1. Установить `Adafruit nRF52 Boards` через `Boards Manager`.
2. Выбрать плату семейства `nRF52840`, совместимую с используемой `ProMicro`.
3. Убедиться, что `USB CDC` включен, если это требуется настройками core.
4. Открыть скетч для `ДУТ` и прошить первую плату.
5. Открыть скетч для `БНСО` и прошить вторую плату.

## Логическое подключение двух плат

Между платами прямое проводное соединение не требуется. Обмен идет по `BLE`.

- `ProMicro #1` подключается к ПК по `USB` как `ДУТ`;
- `ProMicro #2` подключается к ПК по `USB` как `БНСО`;
- управление сценарием выполняется через `Serial Monitor` на стороне `ДУТ`;
- прием телеметрии наблюдается через `Serial Monitor` на стороне `БНСО`.

## Минимальные команды для скетча `ДУТ`

Через `Serial` плата `ДУТ` должна поддерживать команды:

- `help` - показать список команд;
- `status` - вывести текущее состояние;
- `set level 574` - установить уровень;
- `set volume 3210` - установить объем;
- `set fueltemp 185` - установить температуру топлива;
- `set airtemp 223` - установить температуру воздуха;
- `set state stable|refill|drain` - установить режим;
- `set flags 0x0181` - установить флаги качества;
- `burst` - отправить ускоренную серию пакетов.

## Проверка после прошивки

После загрузки двух скетчей нужно убедиться, что:

- `ДУТ` печатает текущую конфигурацию и начинает рекламу BLE;
- `БНСО` находит рекламные пакеты;
- `БНСО` декодирует `sensor_id`, `sequence`, уровень, объем и температуры;
- при смене параметров на `ДУТ` обновления видны на `БНСО`.

### Дополнительная тестовая плата

Полезна для:

- генерации тестовых команд;
- имитации нестабильного радиоэфира;
- проверки реакции на burst-события;
- синтетических сценариев заправки и слива.

## Подключение стенда

### Подключение `ДУТ`

- `USB` платы `ДУТ` подключается к ПК;
- аналоговые входы подключаются к макету измерительного канала;
- входы датчиков температуры подключаются к реальным датчикам или эмуляторам;
- питание измерительного канала подается от лабораторного источника или стабилизированного макета.

### Подключение `БНСО`

- `USB` платы `БНСО` подключается к ПК;
- плата работает как BLE-сканер и пишет журнал приема;
- при необходимости на плату выводятся дискретные статусы: `packet_ok`, `crc_error`, `timeout`.

## Что желательно заложить в макет измерительной части

- переключаемые эквиваленты емкости для имитации разных уровней;
- отдельный канал для медленного долива;
- отдельный канал для быстрого слива;
- возможность ввести шум или дрожание измерения;
- эмуляцию температуры топлива;
- эмуляцию температуры воздуха.

## Режимы испытаний стенда

### Базовая связь

Проверяется:

- загрузка прошивки;
- доступность `USB`;
- запуск BLE advertising;
- видимость `GATT` сервиса;
- прием телеметрии на эмуляторе `БНСО`.

### Тарировка

Проверяется:

- чтение сырых значений;
- запись точек тарировочной таблицы;
- чтение назад записанных точек;
- изменение `calibration_revision`;
- сохранение таблицы в памяти.

### Температурная коррекция

Проверяется:

- реакция на изменение температуры топлива;
- реакция на изменение температуры воздуха;
- применение корректирующих коэффициентов;
- наличие соответствующих флагов в телеметрии.

### Сценарии долива и слива

Проверяется:

- распознавание режима `refill`;
- распознавание режима `drain`;
- отдельные временные константы фильтра;
- отсутствие ложных срабатываний на мелком шуме.

### Протокол `ДУТ -> БНСО`

Проверяется:

- стабильность `manufacturer data`;
- корректность кодирования полей;
- рост `sequence`;
- дедупликация на стороне `БНСО`;
- обработка потери нескольких пакетов подряд.

### Протокол `ДУТ <-> Android`

Проверяется:

- чтение телеметрии;
- чтение диагностики;
- команды `HELLO`, `READ_CAL_POINT`, `WRITE_CAL_POINT`;
- запись параметров долива и слива;
- уведомления о событиях.

## Минимальный порядок запуска

1. Подключить обе `ProMicro` платы к ПК.
2. Залить Arduino-скетч `ДУТ` и Arduino-скетч эмулятора `БНСО`.
3. Проверить, что `ДУТ` виден по `USB`.
4. Проверить, что `ДУТ` публикует BLE-пакеты.
5. Проверить, что `БНСО` принимает и декодирует телеметрию.
6. Подключить Android-приложение к `GATT` сервису.
7. Прогнать базовую тарировку и несколько сценариев изменения уровня.

## Что логировать обязательно

- сырое значение измерительного канала;
- отфильтрованное значение;
- текущую температуру топлива;
- текущую температуру воздуха;
- `level_state`;
- `quality_flags`;
- номер `sequence`;
- ревизию тарировки;
- время прихода пакета на стороне `БНСО`.

## Практические замечания

- на раннем этапе лучше использовать одинаковые `ProMicro` платы, чтобы сократить число платформенных различий;
- платформенно-зависимые пины стоит вынести в отдельный конфиг;
- для первых тестов полезно отделить BLE-отладку от аналоговой части и подменять измерения синтетическими значениями;
- если стенд используется для нескольких сценариев, стоит завести фиксированные профили: `baseline`, `refill`, `drain`, `noisy`.
